# Практична робота 7 (Варіант 21) 

# Проєкт "Кулінарна книга" 

Цей мініпроєкт є десктопним застосунком, створеним на базі **WinUI 3** (Windows App SDK) та мови C#. Основна мета проєкту демонстрація взаємодії з локальною базою даних за допомогою **Entity Framework Core** (провайдер SQLite) та реалізація базових операцій CRUD (Create, Read, Update, Delete).

## Логіка проєкту

Застосунок дозволяє користувачу керувати базою кулінарних рецептів. Дані зберігаються у локальному файлі `cookbook.db`, який автоматично створюється при першому запуску програми. 

Основні можливості:
* **Перегляд (Read):** Усі збережені рецепти відображаються у списку зліва. При виборі рецепту зі списку, його деталі завантажуються у форму справа. Під час завантаження списку використовується жадібне завантаження (Eager Loading) для підтягування пов'язаних категорій.
* **Створення (Create):** Користувач може ввести назву та інструкцію для нового рецепту і зберегти його в базу.
* **Оновлення (Update):** Вибравши існуючий рецепт, користувач може змінити його текст і зберегти оновлення.
* **Видалення (Delete):** Вибраний рецепт можна видалити з бази даних.

## Структура файлів

### 1. `CookbookData.cs` (Моделі та Дані)
У цьому файлі описана структура бази даних та моделі сутностей:
* **`Category`**, **`Recipe`**, **`Ingredient`** — класи (POCO-моделі), що описують таблиці в базі даних та зв'язки між ними (наприклад, один рецепт належить до однієї категорії, але має багато інгредієнтів).
* **`CookbookDbContext`** — клас контексту Entity Framework. Відповідає за налаштування підключення до SQLite (`OnConfiguring`) та містить набори даних (`DbSet`) для кожної таблиці.

### 2. `MainWindow.xaml` (Інтерфейс користувача)
Файл розмітки (XAML), який визначає візуальну частину застосунку.
* Розділений на дві колонки за допомогою `Grid`.
* Ліва частина містить `ListView` для відображення списку рецептів. Використовується прив'язка даних (Data Binding) для виведення назви рецепту та його категорії.
* Права частина містить елементи керування: `TextBox` для введення/редагування тексту та `Button` для виконання дій (Додати, Оновити, Видалити).
* **Важливо:** Усі елементи налаштовані декларативно (відступи, розміри, кольори), тому програмне налаштування UI в C#-коді не використовується.

### 3. `MainWindow.xaml.cs` (Code-Behind / Логіка інтерфейсу)
Клас, який обробляє взаємодію користувача з інтерфейсом та викликає відповідні методи бази даних:
* **`InitializeDatabase()`**: Перевіряє наявність бази даних, створює її за потреби та заповнює початковими (тестовими) категоріями.
* **`LoadData()`**: Виконує запит до `CookbookDbContext`, отримує список рецептів і прив'язує їх до `ListView`.
* **Обробники подій (`AddButton_Click`, `UpdateButton_Click`, `DeleteButton_Click`)**: Містять логіку відкриття підключення (через блок `using`), модифікації даних та виклику `db.SaveChanges()`.
* **`RecipesListView_SelectionChanged`**: Відповідає за перенесення даних вибраного зі списку рецепту в текстові поля форми.

## Використані технології
* C# / .NET
* WinUI 3 (Windows App SDK)
* XAML (для проєктування UI)
* Entity Framework Core (Microsoft.EntityFrameworkCore.Sqlite)
